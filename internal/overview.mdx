## Overview

This section is the internal product map for Zell. The goal is to document every meaningful platform feature in one place so employees can quickly answer how a feature works, where it appears, who can access it, and which backend logic drives it.

Use the feature pages as living outlines. Some pages are complete; others are intentionally lightweight placeholders until the automatic documentation agent fills them from source changes.

## Documentation Standard

Each feature page should eventually answer:

| Question | Meaning |
| --- | --- |
| What is it? | Short product explanation in customer/internal language. |
| Where is it? | Main navigation path, app route, or user surface. |
| Who uses it? | Roles and permission boundaries. |
| How does it work? | Main user workflow and backend behavior. |
| What data drives it? | Important filters, query keys, APIs, and source tables. |
| What can go wrong? | Empty states, edge cases, and support/debug notes. |
