## Overview

Scorecards and signals define how Zell evaluates calls and training interactions. Signals are reusable evaluation criteria; scorecards combine them with weights and assignment rules.

## Key Concepts

<CardGroup cols={2}>
  <Card title="Signal" icon="signal">

  </Card>

  <Card title="Scorecard" icon="star">

## Zell AI Generation

Generated scorecards must contain at least one signal before they are handed to the scorecard creation service. Persisted scorecard metadata can still represent an empty signal list for existing records or intermediate admin edits, but the Zell AI generation path uses a stricter output contract so the model cannot silently create a scorecard without scoring criteria. Structured-output tooling must preserve array shape and item constraints for generated signals, including arrays wrapped in schema defaults.
