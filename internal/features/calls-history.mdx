## Overview

Calls History is the table view inside the user Calls workspace. `/calls` now opens the personal Coaching Insights view by default; users switch to the historical table with `/calls?view=list`. The table shows call metadata, agent information, evaluation results, contact information, and feedback counts so users can inspect past conversations and open detailed call analysis.

## Key Concepts

<CardGroup cols={2}>
  <Card title="User Scope" icon="user">

## Calls Workspace Routing

| URL | View | Frontend component | Notes |
| --- | --- | --- | --- |
| `/calls` | Coaching Insights | `UserCallsWorkspace` renders `UserCoachingInsightsDisplay`. | Default view. Any `view` value other than `list` falls back here. |
| `/calls?view=list` | Table view | `UserCallsWorkspace` renders `UserCallsHistoryDisplay`. | Historical Calls table documented on this page. |

The view toggle is client-side and does not change backend permissions. Clicking a Calls History row routes to `/call-analysis/{callId}`.

### Coaching Score Ranges

| Option | Request fields |
| --- | --- |
| All coaching scores | No `minCoachingScore` or `maxCoachingScore`. |
| High coaching | `minCoachingScore=4`; no maximum. |
| Medium coaching | `minCoachingScore=2&maxCoachingScore=4`. |
| Low coaching | `minCoachingScore=0&maxCoachingScore=2`. |

## API-Supported Filters

Some filters are supported by the API hook or backend even when they are not currently visible in the Calls History table UI.

| Filter | Query parameter | Behavior |
| --- | --- | --- |
| Page | `page` | Defaults to `1`. |
| Page size | `pageSize` | Defaults to `10`. |
| Scorecards | `scorecardIds` | Matches agent scorecards or call metadata scorecards. |
| Date from | `startDate` | Filters `startedAt >= startDate`. |
| Date to | `endDate` | Filters `startedAt <= endDate`. |
| Coaching score min | `minCoachingScore` | Filters to calls in the Zell/coachable score range. |
| Coaching score max | `maxCoachingScore` | Used with the score range filter. |
| Has comments | `hasComments=true` | API hook supports this flag, but `ListCallsSchema` in the local backend does not currently declare it in the OpenAPI query schema. Handler reads only the literal string `true`. |
| Has unread comments | `hasUnreadComments=true` | API hook supports this flag, but `ListCallsSchema` in the local backend does not currently declare it in the OpenAPI query schema. Requires unread coachable feedback without a dislike vote. |
| Source | `source` | Filters by call source, `SIMULATION` or `REAL_CALL`; not visible in the current table view. |

## How It Works

<Steps>
  <Step title="Choose filters">
    The user switches to `/calls?view=list`, then selects scorecards, a coaching score range, and a date range.
  </Step>

  <Step title="Request history">
    The frontend calls `/calls/history` with page, page size, scorecard IDs, date range, and coaching score parameters.
  </Step>

  <Step title="Scope the query">
    The backend uses the authenticated user ID and only returns that user's valid calls.
  </Step>

  <Step title="Enrich rows">
    The service adds agent data, evaluation details, contact information, and feedback counts.
  </Step>

  <Step title="Open call analysis">
    Clicking a row opens the detailed call analysis page for that call.
  </Step>
</Steps>

## Operational Notes

- Query key starts with `['calls-history']`.
- Date range is stored under `calls-history-date-range`.
- Coaching score is stored under `calls-history-coaching-score`.
- Selected scorecards are stored under `calls-history-scorecard-ids`.
- Page number persists in module memory, not local storage; changing page size resets the page to `1`.
- Page size defaults to `10`.
- Calls are ordered by `startedAt` descending.
- The route passes `valid: true`, so invalid calls are excluded.
- If `hide_score` is enabled or unresolved, the frontend hides score display in the table.
