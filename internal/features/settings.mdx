## Overview

Settings control tenant-level and user-level configuration such as language, provider integrations, account defaults, and application preferences.

## Key Concepts

<CardGroup cols={2}>
  <Card title="Tenant Settings" icon="building">

  </Card>

  <Card title="User Settings" icon="user-cog">

  </Card>

  <Card title="Language Settings" icon="languages">

## Frontend Navigation and Roles

The Settings sidebar is built by `src/components/layouts/SettingsNav.tsx`.

| Section | Visible from frontend | Includes |
| --- | --- | --- |
| `Team` | `ADMIN` and `TEAM_LEAD` | Overview, Users, Groups, Workspaces |
| `General` | `ADMIN` only | Usage, Notification preferences, Feedback tone, Custom vocabulary, Call types, Call outcomes |
| `Integrations` | `ADMIN` only | Slack, Aircall, Amplemarket, HubSpot, Gong, tl;dv, and Dialer when the tenant has the `DIALER` feature |

Tenant settings routes are generated by `src/appRoutes.ts` under `/settings/tenant/:tenantId`. The tenant settings root redirects to `/settings/tenant/:tenantId/usage`.

## tl;dv Settings Page

The tl;dv integration settings page is available at `/settings/tenant/:tenantId/tldv` and appears in the `Integrations` section for admins with the sidebar label `tl;dv`.

The page is implemented by `src/app/(auth)/settings/tenant/[tenantId]/tldv/TldvSettingsClient.tsx` and uses `useTldvSettingsViewModel` for connection state, import filters, recent meetings, selection, and import actions. Detailed tl;dv filter and endpoint behavior is documented in [Integrations](/internal/features/integrations#tldv-integration).

Important employee-facing notes:

| Question | Answer |
| --- | --- |
| Who should customers ask to configure tl;dv? | A tenant admin. The frontend only exposes tenant integration settings to `ADMIN` users. |
| Can team leads configure tl;dv from Settings? | Not from the current sidebar. Team leads see Team settings links but not tenant integration links. |
| Is tl;dv gated by the `DIALER` feature flag? | No. Only Dialer is conditionally shown based on `Features.DIALER`; tl;dv is listed for admins regardless of that frontend feature check. |
| Where should support look for filter details? | Use the tl;dv section in the internal Integrations docs. It maps each UI label to frontend state, request fields, defaults, endpoints, null/empty behavior, and source files. |
