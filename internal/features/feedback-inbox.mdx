## Overview

The Feedback Inbox is the daily review surface for reps. It presents unread coaching feedback from the rep's own calls, one snippet at a time, so they can review the feedback, answer lightweight coaching prompts, and continue through their daily plan.

Managers have a separate follow-up inbox for team replies. See [Manager Feedback Inbox](/internal/features/manager-feedback-inbox).

## Key Concepts

<CardGroup cols={2}>
  <Card title="Daily Plan" icon="calendar-days">
    Zell builds a deterministic daily plan for each rep from eligible unread feedback.
  </Card>

  <Card title="Snippet" icon="message-square">
    Each inbox item is a feedback snippet with transcript context, reviewer information, replies, score data, and audio when available.
  </Card>

  <Card title="Interaction Type" icon="mouse-pointer-click">
    Items can be passive review, score estimate, or multiple choice depending on available score and transcript data.
  </Card>

  <Card title="Read Progress" icon="check-check">
    Completing an item marks the feedback as read and moves the rep toward the daily review goal.
  </Card>
</CardGroup>

## How It Works

<Steps>
  <Step title="Open the Home Page">
    Non-manager users see the Feedback Inbox on the authenticated home page.
  </Step>

  <Step title="Load the Current Item">
    The frontend requests `/calls/inbox-snippets`. The backend resolves today's plan and returns the first unread item.
  </Step>

  <Step title="Review the Snippet">
    The rep sees the coaching feedback, transcript context, replies, and audio controls when a recording is available.
  </Step>

  <Step title="Complete the Interaction">
    The rep either completes a passive review, estimates the score, or answers a multiple-choice prompt.
  </Step>

  <Step title="Continue">
    The backend marks the feedback as read. The frontend keeps the reveal state visible until the rep clicks continue, then loads the next item.
  </Step>
</Steps>

## Item Selection

The inbox does not expose manual filters. Zell selects items automatically.

| Rule | Behavior |
| --- | --- |
| Owner | Only feedback from the current rep's own calls is eligible. |
| Time window | Feedback must be from the last 30 days. |
| Read status | Feedback must be unread. |
| Dislike exclusion | Feedback with any dislike vote is excluded. |
| Focus priority | Feedback matching the rep's coaching focus is prioritized. |
| Coachability | Higher-coachability snippets rank earlier within the plan. |
| Plan size | The backend can plan up to 30 items per day. |
| Daily goal | The frontend progress goal is 10 completed items. |

<Note>
For the deeper backend algorithm, see [Snippet Selection Logic](/internal/features/backend-snippet-selection).
</Note>

## Interaction Types

| Type | When it appears | User experience |
| --- | --- | --- |
| Passive review | Always eligible | The rep reads the feedback and continues. |
| Score estimate | The snippet has an AI score | The score is hidden until the rep submits an estimate. |
| Multiple choice | Negative snippet, transcript index, and rep line exist | The rep chooses the best coached response. |

---

## Actions

| Action | What happens |
| --- | --- |
| Reply | Adds a feedback reply and refreshes related inbox, call, snippet, history, manager pending reply, and user stats data. |
| Save for later | Opens the collection dialog for the current call. |
| Continue | Fetches the next unread item from today's plan. |

## Empty State

The inbox is empty when there are no eligible unread feedback snippets or when all items in today's plan are complete.

## Operational Notes

- Query key: `['inbox-snippets']`
- Answer endpoint: `/calls/inbox-snippets/{feedbackId}/answer`
- Answering intentionally does not immediately invalidate the inbox query, so the reveal state remains visible until continue.
- Extra reps are tracked locally with `zell:extra-reps:{yyyy-mm-dd}` after the daily progress goal is reached.
